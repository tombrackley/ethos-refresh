Skills — simple & logical solutions

Skills, stripped back to a human-owned model. The user manages their own skill levels with light AI assistance — the platform surfaces skills and evidence, it doesn't compute a score. CPD compliance stays a separate layer that works exactly as it does today.

1 · Two layers, kept separate

CPD and the Skills Profile answer different questions, are owned differently, and must not be conflated. CPD is rules-driven and stays exactly as it is today. The Skills Profile is human-owned and deliberately simple.

CPD skills · CPD Tracker Compliance. Regime-defined categories, points, deadlines. Tracked exactly as today — this complexity is necessary because the regimes prescribe it.
Skills Profile · Your Skills Matrix Capability. You place each skill at Emerging / Established / Expert. Self-managed with light AI assistance — no weighting, no computed score.

  • CPD = compliance, unchanged. Driven by external rules; the heavy structure belongs there and stays.
  • Skills Profile = capability, user-owned. Because the user sets their own level, the measurement machinery (weights, scoring, rubrics) isn't needed.
  • One thin connection. Completing a CPD activity credits the regime and shows up as supporting evidence on the mapped profile skill — it never auto-changes the level.

Three entry states, because every org is somewhere different on the journey:

Empty

  • Org brought no skills data.
  • Matrix starts blank with an Upload CV prompt.
  • AI parses the CV and suggests skills you keep and place.

Unconfirmed

  • Org assigned skills (role / CV / framework).
  • They show pending — dashed, with ✓ confirm / ✕ reject.
  • Review, move to the right level, confirm. "Accept all" clears the lot.

Established

  • Your confirmed matrix.
  • Add skills any time (manually or via CV).
  • Nudges suggest moves as you complete activity.
  • The skill is shared; the record is yours. "Ethics" is the same canonical concept everywhere; your level, evidence and placement are per-user.
  • CV upload is the fast path. It's how a blank profile becomes useful in one step — Ramona's "first thing I'd do."
  • The org can assign, the user confirms. An admin's assignment is a suggestion until the user reviews it.

5 · Evidence backs a level — it doesn't compute it

Each skill carries evidence: the things that justify where you've placed it. Evidence is supporting context, not a weighted input. It includes work done outside Ethos that the system can't see.

6 · The end-to-end loop

The platform's role is to surface, not to score. The user stays in control of the level at the end.

User does something

  • Completes a CPD activity, assessment or credential
  • Uploads a CV
  • Does relevant work (incl. outside Ethos)

Platform tags & surfaces

  • Maps the activity to a canonical skill
  • Credits the CPD regime (compliance)
  • Adds it as evidence on the profile skill
  • Optionally nudges: "move this up?"

User decides

  • Sets or confirms the level (Emerging / Established / Expert)
  • Accepts, ignores or dismisses a nudge
  • Adds or removes evidence
  • Reviews org-assigned skills

8 · What this model deliberately leaves out

Concepts we removed because the user now owns the level — keeping them would re-introduce the complexity we set out to cut.

  • System-computed / weighted levels. No aggregator, no per-signal weights, no hidden "score". The level is the user's call.
  • "Request assessment" as a scorer. Removed. An assessment can suggest a starting point, but it doesn't set your level.
  • Five-level scales & fine granularity. Three levels is enough when a human is placing them.
  • Role progression %. "72% to Partner" is fiction without a maintained role-skill matrix. Park it.
  • Per-skill numeric targets, peer / industry comparisons. Out of scope until there's an authoritative source.

The test

For any level, badge or evidence row, ask: who put it there — the user, a platform mapping, or a CPD regime? If the honest answer is "the system guessed it from weighted signals," it doesn't belong in this model. What survives is human-owned, traceable, and simple.
