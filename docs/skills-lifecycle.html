Parallel · CPD Regimes

Adjacent system — not in this lifecycle

What CPD regimes are, and why they sit alongside skills

CPD regimes are activity-based compliance requirements set by accreditation bodies — e.g. Law Society NSW requires 10 CPD hours per year including ethics; AICD and AUSTRAC have their own. They're about what learning you must complete in a period, not how capable you are at a competency.

Lives in: CPD Tracker (user-facing) · /admin/cpd-management + /admin/cpd-regimes (admin).

Touchpoints with skills — limited and one-directional:

  • Same bodies attribute both. A library skill cites Law Society NSW; Law Society NSW also defines a CPD regime. Shared citation, not shared data.
  • A course completion can count toward CPD hours and appear as evidence on a skill. One event, two consumers.
  • An org's CPD regime subscriptions might default-enable matching library skills as a convenience — not a coupling.

Per the Skills direction principle "regimes separate": the two systems share source frameworks but don't share data structures or governance.

Who owns what

Tier Owner What they do
Tier 0 · Library Ethika Admin Curates the master skills library. Writes descriptions, sets default categories, ties skills to source frameworks, retires obsolete entries.
Tier 1 · Catalog Org Admin Picks library skills that apply to the org. Adds custom skills not in the library. Manages org-level categories and regime scoping.
Tier 2 · Roles Org Admin For each role: attaches catalog skills, sets target level per skill, writes role description, assigns members.
Tier 3 · Assessment Team Member primary
Org Admin override		 User self-assesses + adds evidence. Admin can edit a cell in the matrix with evidence required. Self-assessment remains the primary signal.
Personal skills Team Member only Anything outside the org catalog. Live on profile only; appear in "All Team Skills" aggregate but never in role gap analysis.

Feedback loops

Library updates → orgs. Ethika revises a skill; subscribed orgs get a "review changes" prompt. Sync is opt-in — nothing forces.
Personal skills → org catalog. When N+ members add the same personal skill, the admin sees "12 team members have added Mediation — promote to org-tracked?" Suggestion only.
Self-assessment → role gap analysis. Ratings feed the matrix gap view; gaps drive training assignment and team-health metrics.
Admin override → audit trail. When the admin edits a cell, evidence is captured. The user can challenge (deferred). Self-assessment stays primary.

What impacts skills

  • Framework changes — ISSB publishes new standards → Ethika updates library → orgs review and sync.
  • Regulator updates — Law Society changes CPD requirements → library skills carry an updated suggested target.
  • Org strategy — Blackmores enters M&A → admin enables M&A skills + adjusts senior-lawyer role targets.
  • Role redesigns — "Senior Lawyer" becomes "Lead Counsel" → admin renames role + revises skills + targets.
  • Team activity — Course completions, scenario assessments, work signals → system suggests level changes; user decides.
  • Bottom-up emergence — Many members share an off-catalog skill → admin sees "promote to catalog?" prompt.
Aligned with: the Skills direction (3-point Emerging / Established / Expert, self-assessed not AI-scored, evidence kept, regimes separate). Built today: /admin/team-capability, /admin/role-profiles. Next: Ethika Library admin (inside EthikaAdminPage), then /admin/skills-catalog.
